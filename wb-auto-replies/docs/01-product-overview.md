# Product Overview

## Задача

Построить систему автоматической работы с отзывами Wildberries для нескольких магазинов:

1. загрузка исторических отзывов и ответов в БД
2. регулярная загрузка новых отзывов
3. формирование ответов по правилам и GPT
4. хранение всех артефактов в БД
5. безопасный запуск через draft/debug этап до production publish

## Scope v1

### Входит

- multi-store support
- ingest archive feedbacks
- ingest active/new feedbacks
- unified normalized storage in PostgreSQL
- classification: karmic vs real
- safe salutation/name guard
- template-based replies for karmic reviews
- GPT-based draft generation for real reviews
- article context from last 10 reviews
- semantic template context for GPT
- photo-aware generation support
- draft mode without publication
- publish mode with audit trail
- health monitoring in DB tables

### Не входит в v1

- полноценная human review web app
- AB testing prompts
- advanced analytics dashboard
- auto-learning from accepted/rejected replies
- omnichannel beyond WB

## Key business rules

### Магазины

- система работает с несколькими магазинами
- у каждого магазина свои токены, настройки, режим запуска и шаблоны

### История и новые отзывы

- архивные и реальные отзывы приходят из разных WB API
- архивные нужно загружать для исторического контекста и аналитики
- новые отзывы нужно регулярно подтягивать несколько раз в день

### Обращение по имени

- имя покупателя анализируется отдельным safety step
- если есть хоть малейшая неуверенность, использовать только: `Здравствуйте!`
- персонализированное обращение допустимо только при высокой уверенности

### Тип 1. Кармические отзывы

- ответ без GPT
- ответ по таблице правил/шаблонов
- логика зависит от числа звезд и настроек магазина

### Тип 2. Реальные отзывы

- ответ формируется GPT
- использовать последние 10 отзывов по этому артикулу
- учитывать стилистику предыдущих ответов
- не допускать повторяемости и ощущения шаблонного ответа
- использовать таблицу типовых ответов, сгруппированных по смыслу
- учитывать медиа (фото)

### Изменения отзывов

- отзыв может быть изменен со временем
- нужно хранить связь старой и новой версии
- система должна понимать версионность и не терять историю

### Безопасный запуск

- обязателен debug/draft сценарий до боевого publish
- публикация не должна включаться до успешной отладки pipeline

## Режимы системы

### backfill

- загрузка истории
- без публикации

### draft

- загрузка новых отзывов
- генерация draft ответов
- без публикации в WB

### publish

- загрузка новых отзывов
- генерация ответов
- публикация в WB

## Success criteria

- данные по отзывам и ответам стабильно загружаются в БД
- для каждого магазина поддерживается независимое состояние и настройки
- кармические отзывы обрабатываются по правилам
- реальные отзывы получают качественный draft с учетом контекста
- система безопасно работает в draft режиме до прод-включения
- health tables позволяют быстро понять состояние pipeline
